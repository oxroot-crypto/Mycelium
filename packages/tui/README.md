# @mycelium/tui

> Mycelium 终端交互界面 — 对标 Claude Code 的沉浸式 TUI 体验

## 架构定位

`@mycelium/tui` 是 Mycelium 的**"面孔"**——用户与系统交互的主要界面。它基于 React 渲染到终端（Ink），提供对标 Claude Code 的实时、沉浸、键盘优先的创作体验。它通过 `@mycelium/orchestrator` 控制所有底层子系统，本身不包含业务逻辑。

```
@mycelium/tui（交互层）
  ↓
@mycelium/orchestrator（业务编排层）
```

## 技术选型

- **框架**：[Ink](https://github.com/vadimdemedes/ink) — React for terminal
- **布局**：[ink-box](https://github.com/vadimdemedes/ink-box) — 终端盒子模型
- **输入**：React 原生 `useInput` + 自定义键盘快捷键管理
- **流式输出**：Ink 的 `Static` 组件 + 自定义流式文本组件
- **主题**：CSS-in-JS 风格终端颜色方案

## 设计原则

1. **键盘优先**：所有操作可通过键盘完成，Vim 风格快捷键（`hjkl` 导航可选）。
2. **实时流式**：LLM 输出逐字显示，不等待完整响应。
3. **非阻塞**：后台任务（审校、摘要生成、知识库更新）不阻塞用户操作。
4. **上下文感知**：底栏始终显示当前上下文——章节、字数、Token、风格、状态。
5. **渐进披露**：默认展示概览，按需深入细节（Dashboard → 写作视图 → 审校报告 → 具体问题）。
6. **终端友好**：所有 UI 在 80×24 最小终端下可用，优先 ASCII art 而非 Unicode。

## 依赖

- `@mycelium/core` — 领域类型、TUI 配置

- `ink`（外部） — React for terminal
- `ink-box`（外部） — 布局组件
- `ink-gradient`（外部，可选） — 渐变色
- `ink-spinner`（外部） — 加载动画
